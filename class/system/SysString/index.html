description:

The SysString class fills the void found in ANSI C for a very common data type: a string of characters. Our SysString class is inherently multi-lingual. It is a set of SysChar objects. The SysString class abstracts the operating-system-specific wide character functions. It also prevents the programmer from having to manipulate buffers of memory; this is all handled automatically. The user needs only to reference a single SysString object, and passing a SysString object to a method is as simple as passing an Integer.

examples:

  • This example shows how to do numeric conversion assignments: 
    SysString num;
       
// setup temporary variables
//
byte dbyte = (long)27;
byte dbyte_v;
       
short dshort = (long)27;
short dshort_v;
       
void* dvoidp = (void*)27;
void* dvoidp_v;
       
// test the byte conversions
//
num.assign(dbyte);
num.get(dbyte_v);
       
if (dbyte != dbyte_v) {
  Console::put(L"Error in assign(byte)");
}
       
// test the short conversions
//
num.assign(dshort);
num.get(dshort_v);
       
if (dshort != dshort_v) {
  Console::put(L"Error in assign(short)");
}
       
// test the pointer conversions
//
num.assign(dvoidp);
num.get(dvoidp_v);
       
if (dvoidp != dvoidp_v) {
  Console::put(L"Error in assign(void*)");
}
  • This example shows how to tokenize a string and count the number of tokens: 
    // declare a string
//
SysString str;

// assign value
//
str.assign(L"   oscar had a heap of apples ");

// declare local variable
//
long pos = 0;

// declare a delimiter string
//
SysString delim(L" ");

// tokenize the string into sub strings
//
SysString sub_str;
while (str.tokenize(sub_str, pos, delim)) {
  sub_str.debug(L"token");
}

// count the number of tokens
//
long tok = str.countTokens(L" ");

if (tok != 6) {
  Console::put(L"Error in countTokens");
}
 notes:

  • The SysString class is modeled after Java's StringBuffer class. The capacity parameter is stored within the SysString object. It is automatically managed by all these methods, but the programmer can exert direct control if desired. The getChars methods are provided as a crutch, they should not be used in most circumstances. All methods in this class operate on the string itself - input SysString objects are left unchanged.

  • While friend functions are generally discouraged in the ISIP environment, they are used in the SysString class to isolate implementation details. Currently, wide character support varies greatly between different operating systems, so we abstract a core group of wide character functions in this class as private methods. The file class also needs to access these wrappers directly for I/O purposes, though, so the friend directive is used.
